\## Зачем нужен двуязычный сайт?



Наличие сайта на нескольких языках делает ваши исследования доступными для большего числа людей. Мой сайт теперь поддерживает:



\- \*\*Английский\*\* - для международной аудитории

\- \*\*Русский\*\* - для местных читателей



\## Как Hugo обрабатывает многоязычный контент



Hugo упрощает создание двуязычных сайтов:



1\. Создайте `content/en/` для английских страниц

2\. Создайте `content/ru/` для русских страниц

3\. Настройте языки в `hugo.toml`

4\. Hugo автоматически добавляет переключатель языков



\## Преимущества



\- Охват более широкой аудитории

\- Улучшение академической видимости

\- Практика технической русской терминологии



\## Выводы



Создание этого двуязычного сайта научило меня:

\- Генераторам статических сайтов (Hugo)

\- Развертыванию на Git и GitHub Pages

\- Созданию контента в Markdown

\- Интеграции академических профилей (ORCID, ResearchGate)
