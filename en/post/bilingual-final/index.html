## Why a Bilingual Website?

Having a website in multiple languages makes your research accessible to more people. My site now supports:

- **English** - For international audiences

- **Russian** - For local readers

## How Hugo Handles Multilingual Content

Hugo makes bilingual sites easy:

1. Create content/en/ for English pages

2. Create content/ru/ for Russian pages

3. Configure languages in hugo.toml

4. Hugo automatically adds language switchers

## Structure

content/ ├── en/ │ ├── _index.md (English homepage) │ ├── skills.md │ ├── experience.md │ └── post/ └── ru/ ├── _index.md (Russian homepage) ├── skills.md ├── experience.md └── post/

Benefits

  • Reach wider audience
  • Improve academic visibility
  • Practice technical Russian terminology

Lessons Learned

Building this bilingual site taught me:

  • Static site generators (Hugo)
  • Git and GitHub Pages deployment
  • Markdown for content creation
  • Academic profile integration (ORCID, ResearchGate)
